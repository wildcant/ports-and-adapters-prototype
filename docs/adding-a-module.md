# Adding a New Module

This guide walks through creating a new module from scratch. Use the `customer` module as a reference implementation.

---

## 1. Register the module key

Add the module to the `Modules` enum in `backend/src/core/utils/modules-definition.ts`:

```typescript
export const Modules = {
  // ...existing modules
  INVENTORY: 'inventory',
} as const
```

This string is the exact key used for `container.resolve(Modules.INVENTORY)`.

---

## 2. Define public types in `core/types/`

Create a subfolder in `backend/src/core/types/<module-name>/` with three files:

**`common.ts`** — Read DTOs and filterable props:

```typescript
import type { BaseFilterable, OperatorMap } from '../common.js'

export type InventoryItemDTO = {
  id: string
  sku: string
  title: string
  stock_quantity: number
  created_at: Date
  updated_at: Date
  deleted_at: Date | null
}

export interface FilterableInventoryItemProps extends BaseFilterable<FilterableInventoryItemProps> {
  id?: string | string[]
  sku?: string | OperatorMap<string>
  title?: string | OperatorMap<string>
}
```

**`mutations.ts`** — Write DTOs:

```typescript
export type CreateInventoryItemDTO = {
  sku: string
  title: string
  stock_quantity: number
}

export type UpdateInventoryItemDTO = {
  sku?: string
  title?: string
  stock_quantity?: number
}
```

**`service.ts`** — Module service interface:

```typescript
import type { FindConfig } from '../common.js'
import type { Context } from '../context.js'
import type { InventoryItemDTO, FilterableInventoryItemProps } from './common.js'
import type { CreateInventoryItemDTO, UpdateInventoryItemDTO } from './mutations.js'

export type IInventoryModuleService = {
  retrieveInventoryItem(id: string, config?: FindConfig<InventoryItemDTO>, context?: Context): Promise<InventoryItemDTO>
  listInventoryItems(filters?: FilterableInventoryItemProps, config?: FindConfig<InventoryItemDTO>, context?: Context): Promise<InventoryItemDTO[]>
  // ...
}
```

**`index.ts`** — Re-exports:

```typescript
export * from './common.js'
export * from './mutations.js'
export * from './service.js'
```

Then add the re-export to `backend/src/core/types/index.ts`:

```typescript
export * from './inventory/index.js'
```

---

## 3. Create the module folder structure

```
backend/src/modules/<name>/
  models/
    <entity>.ts
    index.ts
  repositories/
    <entity>.ts
    index.ts
  services/
    <name>-module-service.ts
    index.ts
  migrations/          ← generated by drizzle-kit
  drizzle.config.ts
  index.ts             ← Module() definition
```

Optional folders (add when needed):
- `types/` — internal-only types that differ from the public contract
- `utils/` — runtime helpers consumed by services

---

## 4. Define models

Each model is a plain Drizzle `pgTable()` in `models/<entity>.ts`:

```typescript
import { sql } from 'drizzle-orm'
import { integer, pgTable, text, timestamp } from 'drizzle-orm/pg-core'

export const inventoryItemTable = pgTable('inventory_item', {
  id: text('id').primaryKey().default(sql`CONCAT('inv_', REPLACE(gen_random_uuid()::text, '-', ''))`),
  sku: text('sku').notNull(),
  title: text('title').notNull(),
  stock_quantity: integer('stock_quantity').notNull().default(0),
  created_at: timestamp('created_at').defaultNow().notNull(),
  updated_at: timestamp('updated_at').defaultNow().notNull(),
  deleted_at: timestamp('deleted_at'),
})

export type InventoryItem = typeof inventoryItemTable.$inferSelect
export type CreateInventoryItem = typeof inventoryItemTable.$inferInsert
```

Rules:
- Every table must have `id`, `created_at`, `updated_at`, and `deleted_at` columns
- ID prefix is defined once here via SQL default — never repeated elsewhere
- `.references()` only for intra-module FKs
- Cross-module columns (e.g., `customer_id`) are plain `text()` with no `.references()`
- Table names: no prefix by default; manually prefix on collision (e.g., `cart_address`)
- Use `pgEnum` for constrained string columns

Re-export from `models/index.ts`:

```typescript
export { inventoryItemTable } from './inventory-item.js'
export type { InventoryItem, CreateInventoryItem } from './inventory-item.js'
```

## 5. Create repositories

Each repository extends `BaseRepository(<table>)` in `repositories/<entity>.ts`:

```typescript
import { BaseRepository } from '../../../core/utils/base-repository.js'
import { inventoryItemTable } from '../models/inventory-item.js'

export class InventoryItemRepository extends BaseRepository(inventoryItemTable) {
  // Add custom queries not covered by the base repository.
  // The base already provides: find, findById, findByIdOrFail,
  // findAndCount, create, createMany, update, delete, softDelete, restore.
}
```

You get full CRUD, filtering (`$eq`, `$in`, `$like`, `$and`, `$or`, ...), pagination (`skip`, `take`), ordering, field selection, and soft-delete filtering for free.

Add custom methods only for queries the base can't express (joins, aggregations, complex subqueries). Use `this.getClient(context)` for transaction-aware queries and `this.table` to reference columns:

```typescript
async findBySku(sku: string, context?: Context) {
  return this.getClient(context)
    .select()
    .from(this.table)
    .where(and(eq(this.table.sku, sku), isNull(this.table.deleted_at)))
}
```

Re-export from `repositories/index.ts`:

```typescript
export { InventoryItemRepository } from './inventory-item.js'
```

## 6. Create the module service

The service is a class with constructor injection in `services/<name>-module-service.ts`:

```typescript
import type {
  Context,
  CreateInventoryItemDTO,
  FilterableInventoryItemProps,
  FindConfig,
  IInventoryModuleService,
  InventoryItemDTO,
  UpdateInventoryItemDTO,
} from '../../../core/types/index.js'
import type { WithTransaction } from '../../../core/utils/with-transaction.js'
import type { InventoryItemRepository } from '../repositories/inventory-item.js'

type InjectedDependencies = {
  inventoryItemRepository: InventoryItemRepository
  withTransaction: WithTransaction
}

export class InventoryModuleService implements IInventoryModuleService {
  private inventoryItemRepository: InventoryItemRepository
  private withTransaction: WithTransaction

  constructor({ inventoryItemRepository, withTransaction }: InjectedDependencies) {
    this.inventoryItemRepository = inventoryItemRepository
    this.withTransaction = withTransaction
  }

  async retrieveInventoryItem(id: string, config?: FindConfig<InventoryItemDTO>, context?: Context) {
    return this.inventoryItemRepository.findByIdOrFail(id, config, context)
  }

  async listInventoryItems(filters?: FilterableInventoryItemProps, config?: FindConfig<InventoryItemDTO>, context?: Context) {
    return this.inventoryItemRepository.find(filters, config, context)
  }

  async createInventoryItems(data: CreateInventoryItemDTO[], context: Context) {
    return this.withTransaction(context, async (ctx) => {
      return this.inventoryItemRepository.createMany(data, ctx)
    })
  }

  async updateInventoryItems(ids: string[], data: UpdateInventoryItemDTO, context: Context) {
    return this.withTransaction(context, async (ctx) => {
      return this.inventoryItemRepository.update(ids, data, ctx)
    })
  }

  async deleteInventoryItems(ids: string[], context: Context) {
    return this.withTransaction(context, async (ctx) => {
      await this.inventoryItemRepository.softDelete(ids, ctx)
    })
  }
}
```

Key patterns:
- Read methods (`retrieve`, `list`) delegate directly to the repository — no transaction needed
- Write methods (`create`, `update`, `delete`) wrap in `this.withTransaction` for atomicity
- `InjectedDependencies` keys must exactly match the keys in the `Module()` definition's `repositories` object, plus `withTransaction` (auto-registered by bootstrap)
- The service implements the interface from `core/types/`

Re-export from `services/index.ts`:

```typescript
export { InventoryModuleService } from './inventory-module-service.js'
```

---

## 7. Wire the module definition

`index.ts` at the module root:

```typescript
import { Module } from '../../core/utils/module.js'
import { Modules } from '../../core/utils/modules-definition.js'
import { InventoryItemRepository } from './repositories/inventory-item.js'
import { InventoryModuleService } from './services/inventory-module-service.js'

export default Module(Modules.INVENTORY, {
  service: InventoryModuleService,
  repositories: {
    inventoryItemRepository: InventoryItemRepository,
  },
})
```

The repository key (`inventoryItemRepository`) must match what the service expects in `InjectedDependencies`.

---

## 8. Register in the bootstrap

Add the module to `backend/src/container.ts`:

```typescript
import inventoryModule from './modules/inventory/index.js'

bootstrapModule(container, inventoryModule)
```

After this, `container.resolve(Modules.INVENTORY)` returns the `InventoryModuleService` singleton.

---

## 9. Create the drizzle config

`drizzle.config.ts` at the module root:

```typescript
import { defineConfig } from 'drizzle-kit'
import { env } from '../../env.js'

export default defineConfig({
  schema: './src/modules/inventory/models/*.ts',
  out: './src/modules/inventory/migrations',
  dialect: 'postgresql',
  dbCredentials: {
    url: env.SUPABASE_DATABASE_URL,
  },
})
```

---

## 10. Generate and run migrations

```bash
# Generate migration from model changes
npx drizzle-kit generate --config=src/modules/inventory/drizzle.config.ts

# Run migration against the database
npx drizzle-kit migrate --config=src/modules/inventory/drizzle.config.ts
```

- [ ] Repository extending `BaseRepository(<table>)`
- [ ] Service implementing the interface from `core/types/`
- [ ] `Module()` definition with matching repository keys
- [ ] Registered in `container.ts` via `bootstrapModule()`
- [ ] `drizzle.config.ts` configured
- [ ] Migration generated and run
